# Contributing Guidelines

Thank you for your interest in contributing to **IITM BS Support**.  
We welcome contributions that improve functionality, accuracy, usability, maintainability, or documentation quality.

---

## Areas of Contribution

Contributors may assist in the following areas:
- Documentation enhancements
- User interface and user experience improvements
- Bug fixes and edge-case resolution
- Optimization of academic calculation logic
- Development of new tools aligned with the IIT Madras BS curriculum

---

## Contribution Workflow

1. Fork the repository to your GitHub account.
2. Clone your fork locally.
3. Create a feature-specific branch:
   ```bash
   git checkout -b feature/concise-feature-name
4. Implement and test your changes.

5. Commit changes with a clear and descriptive message.

6. Push the branch to your fork.

7. Open a Pull Request against the main repository.

## Pull Request Requirements

Keep pull requests focused and limited in scope.

Clearly describe the purpose, motivation, and impact of the changes.

Reference relevant issues when applicable.

Ensure existing functionality remains intact.

Include documentation updates where appropriate.

## Development Standards

Follow the established project structure and naming conventions.

Write clean, readable, and maintainable code.

Avoid introducing unnecessary dependencies.

The project must remain fully client-side unless changes are discussed beforehand.

## ommunication Guidelines

If you are uncertain about a proposed change:

Open an issue to initiate discussion, or

Comment on an existing issue before beginning implementation.

Clear communication ensures efficient collaboration and avoids redundant work.

## Code of Conduct

All contributors are expected to engage respectfully and professionally.
This project is committed to maintaining an inclusive, collaborative, and supportive open-source environment.
