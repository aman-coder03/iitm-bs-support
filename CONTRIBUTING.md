# Contributing Guidelines

Thank you for your interest in contributing to IITM BS Support.  
Contributions that improve functionality, documentation, usability, or accuracy are welcome.

---

## Ways to Contribute

You can contribute by:
- Improving documentation
- Enhancing UI or UX
- Fixing bugs
- Improving calculation accuracy
- Adding new academic tools relevant to the IIT Madras BS program

---

## Getting Started

1. Fork the repository to your GitHub account.
2. Clone your fork locally.
3. Create a new branch for your changes:

        git checkout -b feature/your-feature-name

4. Make your changes.
5. Commit with a clear and descriptive message.
6. Push the branch to your fork.
7. Open a Pull Request to the main repository.

---

## Pull Request Guidelines

- Keep pull requests focused and minimal.
- Clearly describe what the change does and why it is needed.
- Reference related issues where applicable.
- Ensure changes do not break existing functionality.
- Documentation updates are encouraged alongside code changes.

---

## Code Style and Practices

- Follow existing project structure and naming conventions.
- Keep logic simple and readable.
- Avoid introducing unnecessary dependencies.
- Ensure the project remains fully client-side unless discussed beforehand.

---

## Communication

If you are unsure about a change, open an issue or comment on an existing one before starting work.  
Clear communication helps avoid duplicate or conflicting contributions.

---

## Code of Conduct

Be respectful and constructive in discussions and reviews.  
This project aims to be inclusive and collaborative.
